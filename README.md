# Assignment 39: GenAI App Deployment

## Problem Statement

In this assignment, the goal is to deploy a Generative AI application so that users can interact with it through a web interface.

The application should be deployed using two different platforms:

- Streamlit Community Cloud
- Hugging Face Spaces

The purpose of this assignment is to understand real-world deployment workflows for Generative AI applications.

---

## Project Overview

I developed a simple **GenAI-based PDF Question Answering application** using Streamlit and LangChain.

The application allows users to:

1. Upload a PDF document
2. Ask questions related to the content
3. Receive answers generated by an LLM based on the document context

The system uses a **Retrieval Augmented Generation (RAG)** approach.

---

## Technologies Used

- Python
- Streamlit
- LangChain
- HuggingFace Models
- Sentence Transformers
- FAISS Vector Database
- PyPDF

---

## How the Application Works

1. The user uploads a PDF file.
2. The PDF is loaded and text is extracted.
3. The extracted text is split into smaller chunks.
4. Each chunk is converted into embeddings using a sentence transformer model.
5. These embeddings are stored in a FAISS vector database.
6. When a user asks a question, similarity search is used to retrieve the most relevant chunks.
7. The retrieved chunks are sent to an LLM which generates the final answer.

## Challenges Faced

During the development and deployment process, I encountered several challenges.

### Python Version Compatibility

Initially, the deployment failed because the environment was using **Python 3.13**, which caused compatibility issues with some libraries like `tokenizers`.  

This was resolved by specifying **Python 3.11** in the `runtime.txt` file.

---

### Dependency Installation Issues

Some packages required specific versions to install correctly in the deployment environment.  
To solve this, I carefully configured the `requirements.txt` file to include compatible library versions.

---

### Large Library Installations

Libraries such as `torch` and `transformers` are large and can slow down deployment.  
Using CPU-based torch installation helped reduce deployment issues.

---

### Environment Differences

Sometimes the application worked locally but failed in cloud deployment due to environment differences.  
This required debugging the dependency versions and ensuring the same setup works in both environments.

## Comparison of Deployment Platforms

### Streamlit Community Cloud

Pros:
- Very easy to deploy
- Direct GitHub integration
- Good for data apps and dashboards

Cons:
- Limited compute resources
- No GPU support

---

### Hugging Face Spaces

Pros:
- Designed for AI/ML applications
- Supports GPU
- Good integration with HuggingFace models

Cons:
- Slightly more complex setup
- Build process can take longer

---

## When to Use Each Platform

**Streamlit Community Cloud**

Best for:
- Data dashboards
- Simple AI applications
- Rapid prototypes

**Hugging Face Spaces**

Best for:
- ML model hosting
- AI demos
- LLM applications
